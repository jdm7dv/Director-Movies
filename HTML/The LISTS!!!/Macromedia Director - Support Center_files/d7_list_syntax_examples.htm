Product: Director Platform: All Versions: 7 and above ID: 14109

Creating new list variables
The following commands are used in order to create new list variables.

return to top

"[ ]":

traditional: someVariableName = []
traditional: someVariableName = [someValue1, someValue2,...]
traditional: someVariableName = [:]
traditional: someVariableName = [#someProperty1:someValue1,#someProperty2:someValue2,...]
dot-syntax: There is no new syntax for this command.

Example: The following command creates a new empty linear list variable named someLinearList.

-- traditional
someLinearList = []

Example: The following command creates a new linear list variable named someLinearList and populates the list with values. The resulting list is [24,36,79,12].

-- traditional
someLinearList = [24,36,79,12]

Example: The following command creates a new empty property list variable named somePropertyList.

-- traditional
somePropertyList = [:]

Example: The following command creates a new property list variable named somePropertyList and populates the list with properties and values. The resulting list is [#a: 98, #b: 112, #c: 9, #d: 49].

-- traditional
somePropertyList = [#a: 98, #b: 112, #c: 9, #d: 49]

return to top

"duplicate":

traditional: duplicate(someListVariable)
dot-syntax: someListVariable.duplicate()

Examples: The following commands create a new list variable named myNewList which is a duplicate copy of the linear list myList. The resulting value of myNewList is ["Sally","Steve","Jen","Fred"].

-- traditional
myList = ["Sally","Steve","Jen","Fred"]
myNewList = duplicate(myList)

-- dot-syntax
myList = ["Sally","Steve","Jen","Fred"]
myNewList = myList.duplicate()

return to top

"list":

traditional: list(someValue1,someValue2,...)
dot-syntax: There is no new syntax for this command.

Example: The following command creates a new linear list variable named someLinearList. The resulting list is ["red","green","blue"].

-- traditional
someLinearList = ["red","green","blue"]

return to top

Adding values and properties to a list variable
The following commands are used in order to add new values or new properties to an already existing list variable.

Return to top

"add":

traditional: add someLinearList,someValue
dot-syntax: someLinearList.add(someValue)

Examples: These statements add the value 80 to the unsorted linear list named hiScores. The resulting list is [98,49,72,80].

-- traditional
hiScores = [98,49,72]
add hiScores,80

-- dot-syntax
hiScores = [98,49,72]
hiScores.add(80)

Note: When using this command with unsorted lists, the value is added to the end of the list. When using this command with sorted lists, the value is added into its appropriate sorted location.

Return to top

"addAt":

traditional: addAt someLinearList,somePositionIndex,someValue
dot-syntax:someLinearList.addAt(somePositionIndex,someValue)

Examples: These statements add the value "dog" to the third position of the unsorted linear list named animals. The resulting list is ["horse","cat","dog","bird"].

-- traditional
animals = ["horse","cat","bird"]
addAt animals,3,"dog"

-- dot-syntax
animals = ["horse","cat","bird"]
animals.addAt(3,"dog")

return to top

"addProp":

traditional: addProp somePropertyList,#someProperty,someValue
dot-syntax: somePropertyList.addProp(#someProperty,someValue)

Examples: These statements add the property #myFavoriteColor and its assigned value of "blue" to the unsorted property list myAttributes. The resulting list is [#myAge: 15,#myFavoriteFood: "hamburgers",#myName: "Joe",#myFavoriteColor: "blue"].

-- traditional
myAttributes = [#myAge: 15,#myFavoriteFood: "hamburgers",#myName: "Joe"]
addProp myAttributes,#myFavoriteColor,"blue"

-- dot-syntax
myAttributes = [#myAge: 15,#myFavoriteFood: "hamburgers",#myName: "Joe"]
myAttributes.addProp(#myFavoriteColor,"blue")

Note: When using this command with unsorted lists, the property is added to the end of the list. If using this command with sorted lists, the property is added into its appropriate sorted location.

Return to top

"append":

traditional: append someLinearList,someValue
dot-syntax: someLinearList.append(someValue)

Examples: These statements add the value 8 to the end of the sorted linear list myValues. The resulting list is [3,9,12,8].

-- traditional
myValues = [3,9,12]
append myValues,8

-- dot-syntax
myValues = [3,9,12]
myValues.append(8)

return to top

Setting values within a list variable
The following commands are used in order to set the values contained by an existing list variable.

return to top

"setaProp":

traditional: setaProp somePropertyList,#someProperty,someValue
dot-syntax: somePropertyList.setaProp(#someProperty,someValue)
dot-syntax: somePropertyList[#someProperty] = someValue

dot-syntax: somePropertyList.someProperty = someValue

Examples: These statements set the value of the property #yesVotes to 38 in the property list voteResults. The resulting list is [#yesVotes: 38,#noVotes: 19].

-- traditional
voteResults = [#yesVotes: 25,#noVotes: 19]
setaProp voteResults,#yesVotes,38

-- dot-syntax
voteResults = [#yesVotes: 25,#noVotes: 19]
voteResults.setaProp(#yesVotes,38)

-- dot-syntax
voteResults = [#yesVotes: 25,#noVotes: 19]
voteResults[#yesVotes] = 38

-- dot-syntax
voteResults = [#yesVotes: 25,#noVotes: 19]
voteResults.yesVotes = 38

Examples: These statements add the property #abstaining to the property list voteResults and set its value to 10. The resulting list is [#yesVotes: 38,#noVotes: 19,#abstaining: 10].

-- traditional
voteResults = [#yesVotes: 38,#noVotes: 19]
setaProp voteResults,#abstaining,10

-- dot-syntax
voteResults = [#yesVotes: 38,#noVotes: 19]
voteResults.setaProp(#abstaining,10)

-- dot-syntax
voteResults = [#yesVotes: 38,#noVotes: 19]
voteResults[#abstaining] = 10

Note: The property must already exist in the list in order to use the dot operator to manipulate it. If the dot operator is used to manipulate a property that does not already exist in the list, a script error will result.

return to top

"setAt":

traditional: setAt someList,someIndex,someValue
dot-syntax: someList.setAt(someIndex,someValue)
dot-syntax: someList[someIndex] = someValue

Examples: These statements set the value of the fourth position in the property list myData to 22. The resulting list is [#testA: 32,#testB: 19,#testC: 7,#testD: 22,#testE: 82].

-- traditional
myData = [#testA: 32,#testB: 19,#testC: 7,#testD: 0,#testE: 82]
setAt myData,4,22

-- dot-syntax
myData = [#testA: 32,#testB: 19,#testC: 7,#testD: 0,#testE: 82]
myData.setAt(4,22)

-- dot-syntax
myData = [#testA: 32,#testB: 19,#testC: 7,#testD: 0,#testE: 82]
myData[4] = 22

Examples: These statements set the value of the seventh position in the linear list myData to 99. The resulting list is [32,19,7,22,82,0,99].

-- traditional
myData = [32,19,7,22,82]
setAt myData,7,99

-- dot-syntax
myData = [32,19,7,22,82]
myData.setAt(7,99)

-- dot-syntax
myData = [32,19,7,22,82]
myData[7] = 99

Note: These commands replace the value currently held within the list variable at the specified position. If a list position is specified that is greater than the number of items in a linear list, Director will expand the list's blank entries to provide the number of places specified by the index value. If a list position is specified that is greater than the number of items in a property list a script error will result.

Return to top

Deleting values and properties from a list variable
The following commands are used in order to delete values or properties from an already existing list variable.

Return to top

"deleteAll":

traditional: deleteAll someList
dot-syntax: There is no new syntax for this command.

Example: These statements delete all items from the linear list testScores. The resulting list is [].

-- traditional
testScores = [99,82,90,71,66,87]
deleteAll testScores

Note: This command deletes all items in the specified list without changing the list type. Therefore, if a deleteAll command is issued for a property list, the resulting list is [:].

return to top

"deleteAt":

traditional: deleteAt someList,someIndex
dot-syntax: someList.deleteAt(someIndex)

Examples: These statements delete the value at the second position in the linear list conxnInfo. The resulting list is ["Sally", "123.456.78.910"].

-- traditional
conxnInfo = ["Sally", "letMeIn!", "123.456.78.910"]
deleteAt conxnInfo,2

-- dot-syntax
conxnInfo = ["Sally", "letMeIn!", "123.456.78.910"]
conxnInfo.deleteAt(2)

Examples: These statements delete the value at the second position in the linear list conxnInfo. The resulting list is [#userName: "Sally",#conxnIP: "123.456.78.910"].

-- traditional
conxnInfo = [#userName: "Sally",#userPass: "letMeIn!",#conxnIP: "123.456.78.910"]
deleteAt conxnInfo,2

-- dot-syntax
conxnInfo = [#userName: "Sally",#userPass: "letMeIn!",#conxnIP: "123.456.78.910"]
conxnInfo.deleteAt(2)

return to top

"deleteOne":

traditional: deleteOne someList,someValue
dot-syntax: someList.deleteOne(someValue)

Examples: These statements delete the value "Tom" from the linear list greenGroupMembers. The resulting list is ["Tim","Sally","Phil","Jennifer"].

-- traditional
greenGroupMembers = ["Tim","Sally","Tom","Phil","Jennifer"]
deleteOne greenGroupMembers,"Tom"

-- dot-syntax
groupLeaders = ["Tim","Sally","Tom","Phil","Jennifer"]
groupLeaders.deleteOne("Tom")

Examples: These statements delete the value "Joe" and its associated value from the property list groupLeaders. The resulting list is [#greenGroup: "Tim", #redGroup: "Nancy"] .

-- traditional
groupLeaders = [#greenGroup: "Tim", #purpleGroup: "Joe", #redGroup: "Nancy"]
deleteOne groupLeaders,"Joe"

-- dot-syntax
groupLeaders = [#greenGroup: "Tim", #purpleGroup: "Joe", #redGroup: "Nancy"]
groupLeaders.deleteOne("Joe")

Note: If there is more than one property within a list variable containing the specified value, then only the first instance of that value and its associated property will be deleted.

Return to top

-- dot-syntax

"getaProp":

traditional: getaProp(someList,somePropertyOrIndex)
dot-syntax: someList.getaProp(somePropertyOrIndex)
dot-syntax: someList[somePropertyOrIndex]
dot-syntax: somePropertyList.someProperty

Examples: These statements set the value of myColor to the value of the property #Henry within the property list favoriteColors. The resulting value of myColor is "red".

-- traditional

myColor = getaProp(favoriteColors,#Henry)

-- dot-syntax
favoriteColors = [#Henry: "red", #Allison: "blue", #Jay: "orange"]
myColor = favoriteColors.Henry

-- dot-syntax

-- traditional

ilkType = ilk(phoneNumbers)

-- dot-syntax
phoneNumbers = [5551234,5554321,5559876]
ilkType = phoneNumbers.ilk

Examples: The following statements attempt to verify that phoneNumbers is a property list by setting ilkCheck to a Boolean value (true or false) based on the type comparison. The resulting value of ilkCheck is 1 or true.

-- traditional
phoneNumbers = [#Joe: 5551234, #Fred: 5554321, #Alan: 5559876]
ilkCheck = ilk(phoneNumbers,#propList)

-- dot-syntax
phoneNumbers = [#Joe: 5551234, #Fred: 5554321, #Alan: 5559876]

Examples: The following statements attempt to verify that phoneNumbers is a list by setting ilkCheck to a Boolean value (true or false) based on the type comparison. The resulting value of ilkCheck is 1 or true.

-- traditional

ilkCheck = ilk(phoneNumbers,#list)

-- dot-syntax
phoneNumbers = [5551234,5554321,5559876]
ilkCheck = phoneNumbers.ilk(#list)

Note: When the ilk command is used to check if a variable is of the type #list, a true result for both property and linear lists is returned. If it is desired to verify that a variable is a linear list, try checking first for a true result using the type #list and then a false result using the type #propList. In addition, this command checks for data types other than lists. Please refer to the Lingo Dictionary for information regarding the other data structure types relating to this command.

Manipulating a list variable

return to top

"sort":

traditional: sort someListVariable
traditional: sort(someListVariable)
dot-syntax: there is no new syntax for this command

Example: The following statements sort the property list groupInfo. The resulting list is [#alpha: ["Tom","Alan"], #beta: ["Don","Tim"], #gamma: ["Ed","Sherry"]].

-- traditional
groupInfo = [#beta: ["Don","Tim"],#gamma: ["Ed","Sherry"],#alpha: ["Tom","Alan"]]
sort groupInfo

Example: The following statement sort the linear list highScores. The resulting list is [63,77,80,92,99].

-- traditional
highScores = [92,99,63,80,77]
sort highScores

Example: The following statements sort the property list groupInfo. The resulting list is [#alpha: ["Tom","Alan"], #beta: ["Don","Tim"], #gamma: ["Ed","Sherry"]].

-- traditional
groupInfo = [#beta: ["Don","Tim"],#gamma: ["Ed","Sherry"],#alpha: ["Tom","Alan"]]
sort(groupInfo)

Example: The following statements sort the linear list highScores. The resulting list is [63,77,80,92,99].

-- traditional
highScores = [92,99,63,80,77]
sort(highScores)
