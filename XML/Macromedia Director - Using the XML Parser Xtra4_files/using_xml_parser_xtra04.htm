Using XML document nodes

The XML document itself can contain different types of nodes. Each node can contain different kinds of data, depending on the node type. You should check the node type before accessing its data so you know what type of data to expect. Nodes are read-only, so you can retrieve the type, but you cannot set it.

Note: The subfield count exists for any field accessible with bracket access. Thus, you can specify whichNode.child.count to find out how many children the specified node has.

Using this XML document as a starting point, the following examples demonstrate how to use these Lingo terms to access the data within various node levels of the XML structure.

The XML looks like this: 

<?xml version="1.0"?>
	<e1>
		<tagName attr1="val1" attr2="val2"/>
		<e2>	element 2</e2>
		<e3>element 3</e3>
		Here is some text
	</e1>

This Lingo returns the name of the first XML tag: 

put gParserObject.child[1].name
-- "e1"

The variable gParserObject contains the parsed XML. When used in the preceding Lingo, it refers to the root node of the XML document. The Lingo term child[1] refers to the first level of nested tag, which is the tag <e1>.

To find out what kind of node the first tag is, use the type function. 

put gParserObject.child[1].type
-- #element

To refer to nodes that are nested more than one level deep, use more than one level of child reference. This Lingo returns the name of the first tag that is nested within the tag <e1>: 

put gParserObject.child[1].child[1].name
-- "tagName"

This Lingo returns the name of the second tag that is nested within the tag <e1>: 

put gParserObject.child[1].child[2].name
-- "e2"

To refer to text data that occurs within a particular tag, use the text property. The text itself is a child node of the tag that contains it, so an additional level of child reference is needed. This Lingo returns the string that appears inside the tag <e2> from the preceding XML example: 

put gParserObject.child[1].child[2].child[1].text
-- "element 2"

In this example, the variable gParserObject refers to the root node of the XML. The child[1] refers to the tag <e1>, which occupies the first level down in the XML's nested structure. The child[2] refers to the second tag within the <e1> tag, which is the tag <e2>. The last child[1] refers to the text within the tag <e2>, which is element 2. Finally, the text property is specified, so that Lingo returns the text of the node rather than any other property of the node.

The fourth child of the tag <e1> is a line of text that reads here is some text. This text is a child just like the XML tags that precede it. You can get the type of this child the same way you would other children.

This Lingo returns the type of the fourth child of the tag <e1>: 

put gParserObject.child[1].child[4].type
-- #text

This Lingo returns the text of the fourth child of the tag <e1>: 

put gParserObject.child[1].child[4].text
-- "
		here is some text
	"

Note that the text element includes the white space for RETURN, SPACE, and TAB characters as well as the string "here is some text".

You can use the Lingo count function to determine the number of children that exist at a particular level if the XML structure. This Lingo returns the number of children at the 2nd level in the preceding XML sample: 

put gparser.child[1].child.count
-- 4
